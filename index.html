¿Todos los recursos tienen los mismos parámetros de configuración?

No, y si bien la tendencia es tratar de homogeneizar estos para que el esfuerzo del diseñador sea mínimo al momento de adaptarlos, la naturaleza de los mismos recursos implica diferencias sustanciales en la naturaleza de su funcionamiento.

¿Todos los recursos están disponibles tanto en escritorio como en móvil?

No, la naturaleza visual y funcional de ciertos recursos dificulta la funcionalidad para el usuario final, por lo que su uso está limitado. De hecho los recursos que por razones de diseño no son aptos para usarse en celular notifican al usuario esta restricción.

¿Como sé cuál versión de recurso estoy usando?

En el archivo index.html de cada recurso, en la parte superior está el historial de versiones, el cual indica, versión, autor, fecha y la descripción de los cambios de esa versión.

¿Qué debo hacer si el reactivo acepta más de una respuesta?

Dirígete al archivo listaReactivos.js y en el apartado de respuestas (A) coloca las respuestas separadas por la  barra vertical | 

Por ejemplo
{ Q: 'Se conoce como @ a la persona que @ la cobertura del seguro y cumple con los requisitos establecidos por la compañía @.',

A: ["respuestaA1|respuestaA2|respuestaA3", "respuestaB1", "respuestaC1|respuestaC2"],

...
Donde el conjunto de respuestas (respuestaA1, respuestaA2, respuestaA3) corresponden al primer @ del reactivo, y así sucesivamente.
¿Qué necesito modificar para que se visualice una cantidad específica de reactivos por página/segmento?

Por ejemplo:
El banco de reactivos contiene 25 reactivos, de los cuales se necesita que se muestren al usuario únicamente 10, y que los pueda visualizar en grupos de tres.

Dirígete al archivo configRecurso.js, y modifica dos variables:

elementosPorSegmento: con esta variable se determina el número de reactivos que quieres visualizar por página/segmento

reactivosMostrar: escribes el total de reactivos que la actividad o autoevaluación podrá mostrar.

Por lo tanto para el ejemplo antes mencionado quedaría

var elementosPorSegmento = 3;

var reactivosMostrar = 10;

¿Cómo y en qué archivo se establecen las retroalimentaciones generales, de acuerdo a ciertos rangos de evaluación?

Dirígete al archivo configRecurso.js. Existe una variable llamada retroCal

var retroCal = [
{LimInf: 0, LimSup: 3, Mensaje: ["No fue suficiente", "Insufficient"]},
{LimInf: 4, LimSup: 6, Mensaje: ["Esfuérzate más", "Work harder"]},
{LimInf: 7, LimSup: 9, Mensaje: ["Suficiente", "Sufficient"]},
{LimInf: 10, LimSup: 10, Mensaje: ["Excelente", "Excellent"]}
];
Donde de acuerdo a los criterios de evaluación se colocan los valores correspondientes a los límites inferiores (LimInf) y los límites superiores (LimSup), así como el respectivo mensaje de retroalimentación, el cual está divido en dos partes: Mensaje: ["Excelente", "Excellent"] para dividir los mensajes en español y los mensajes en inglés.
Cuando una actividad cuenta con múltiples intentos, ¿cómo se pueden conservar las respuestas correctas del intento previo?

Por su naturaleza, sólo algunos recursos soportan el poder conservar las respuestas correctas del intento previo, y son

  • Completar arrastrando
  • Completar arrastrando tabla
  • Completar arrastrando esquema
  • Completar arrastrando audio-video
  • Completar eligiendo
  • Completar escribiendo
  • Opción múltiple
  • Verdadero-falso
  • Ordenar

Dirígete al archivo configRecurso.js. Existe una variable llamada siguienteIntentoBlanco

Para conservar las respuestas correctas, la variable debe tener valor "false"

var siguienteIntentoBlanco = false;

En caso de que no se tengan que conservar las respuestas correctas, la variable debe tener valor "true"

var siguienteIntentoBlanco = true;

En el recurso Completar escribiendo, ¿hay reglas para la escritura de las respuestas?

En estricto sentido, las respuestas se calificarán como correctas según aparezcan en el guión instruccional, esto es, con mayúsculas, minúsculas, tildes y diéresis. Esto está establecido en una variable llamada revisionEstricta que se encuentra en el archivo configRecurso.js. Debe tener el valor "true".

var revisionEstricta = true;

Si se considera que la forma de las respuestas puede ser más flexible, la variable debe tener el valor "false".

var revisionEstricta = false;

¿Para qué se utiliza la variable esParrafo en el recurso Sopa de letras?

Se utiliza cuando los reactivos en la sopa de letras está conformado por una frase y se necesite encontrar la palabra. Para esto, dirígete al archivo configRecurso.js

Existe una variable llamada esParrafo que debe estar con el valor de "true".

var esParrafo = true;

Si se necesita que solo se busquen palabras en la sopa de letras, la variable esParrafo debe tener el valor de "false".

var esParrafo = false;

¿Cómo activar el temporizador?

En cualquiera de los recursos lúdicos, dirígete al archivo configRecurso.js

Existe una variable llamada tempo que debe estar con el valor de "true". Para especificar el tiempo que se necesita, cambia los valores de las variables de minutes y seg

var tempo = true;

var minutes = 1;

var seg = 45;

¿Cómo utilizar el conteo de movimientos en el recurso de Memorama?

Si se desea que haya un conteo de movimientos, en el archivo configRecurso.js se debe colocar la siguiente variable en "true"

var conteoMovimientos = true;

Se puede establecer cuál es el número máximo de movimientos permitidos con la siguiente variable

var movimientos = 10;

NOTA: Si no se establece un número de movimientos, el recurso automáticamente lo establecerá.

En el caso de que se quiera permitir un número de movimientos indefinidos, hay que establecer la variable en "false"

var conteoMovimientos = false;
